# Live Memory: THEN Project - Plan 0 (Prototype)

**STATUS: Code architecture complete. No model has been trained. See docs/plan/early_validation_and_scaling_plan.md for the actual validation path.**

**Goal**: Validate the **THEN (Temporal History Episodic Network)** architecture by demonstrating that a frozen model can ingest and recall episodic memories from a specific dataset (Cairo Episodes) without weight updates.

**Philosophy**: "Ingest, Don't Train." The episodic memory system should allow the model to learn new events instantly by storing them in its internal state, simulating the hippocampus, rather than requiring slow backpropagation training.

## Phase 1: Pretraining (The "Meta-Learning" Phase)

*Goal: Teach the model **how** to use its memory modules.*

Since we cannot train on the Cairo episodes (that would be cheating/baking it in), we must pretrain the model on a generic dataset to learn the mechanics of `HybridTHENAttention`.

1. **Dataset**: Use `FineWeb-Edu` (or a smaller subset for prototyping).
2. **Task**: Standard Next-Token Prediction.
3. **Mechanism Learning**:
    * The model learns to use KDA (Compression) to store information in `state['traces']`.
    * The model learns to use DSA (Retrieval) to query `state['traces']` to predict future tokens.
    * *Note for Prototype*: We might need a "Memory Gym" dataset where the answer *always* requires looking back N tokens, forcing the model to rely on the memory module.
4. **Outcome**: A `pretrained_then.pt` model with frozen weights. **(NOT YET PRODUCED — no training has been run)**

## Phase 2: Ingestion (The "Live" Phase)

*Goal: Populate the memory state with specific episodes.*

1. **Input**: `synthetic-cairo-episodes.txt` (generated by `dev/gen_cairo_data.py`).
2. **Process**:
    * Load `pretrained_then.pt`.
    * Set model to **Eval Mode** (Frozen Weights).
    * Run the Cairo episodes through the model.
    * **Crucial**: Pass the `state` dictionary from one batch to the next.
    * The `HybridTHENAttention` layers will naturally populate `state['traces']` with the Cairo events.
3. **Outcome**: A populated `cairo_memory_state.pt` file (tensor dictionary). **(NOT YET PRODUCED)**

## Phase 3: Recall & Query

*Goal: Demonstrate "Live Memory".*

1. **Input**: User query (e.g., "What coffee did I want at t=7?").
2. **Process**:
    * Load `pretrained_then.pt`.
    * Load `cairo_memory_state.pt`.
    * Run inference. The model should use DSA to retrieve the answer from the loaded state.
3. **Success Metric**: Correct answer without the model ever having *trained* on the Cairo text. **(NOT YET VALIDATED)**
